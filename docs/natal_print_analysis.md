1. **12 个问题** → 需要 **11 个打印板块**（其中 8 个在原局 section，3 个在流年 section）

## 四、打印内容冗杂分析

### 4.1 冗杂问题

| 问题 | 描述 | 建议 |
|------|------|------|
| **性格重复打印** | 主要性格、其他性格、性格快速汇总有重复 | 考虑合并或分层标注 |
| **用神信息过多** | 用神五行、用神→十神、用神落点、六合、天干五合都在一起 | 拆分为"用神核心信息"和"用神 Debug 信息" |
| **原局/大运混杂** | 原局板块里打印了大运转折点、用神互换、大运快照 | 移到大运 section |
| **六亲助力过于详细** | 每个大类都打印来源、强度话术，内容很长 | LLM 可能不需要这么详细的来源描述 |

### 4.2 建议改进

1. **分层标注**：
   - `[LLM]` 标记：LLM 需要的内容
   - `[DEBUG]` 标记：Debug 用的内容
   - `[USER]` 标记：用户想看的基础信息

2. **移除原局 section 中的非原局内容**：
   - 大运转折点 → 移到大运 section
   - 用神互换 → 移到大运 section
   - 大运快照 → 移到大运 section

3. **简化 Debug 信息**（可选）：
   - 用神（五行→十神）可以折叠
   - 用神落点可以折叠
   - 原局六合/半合/天干五合可以合并

## 八、打印风格冗杂问题总结

### 8.1 当前冗杂的地方

| 问题 | 描述 | 建议 |
|------|------|------|
| **性格重复** | 主要性格、其他性格、性格快速汇总有重复内容 | 保留主要性格 + 性格快速汇总，其他性格可折叠或标注为 DEBUG |
| **用神信息过多** | 用神五行、用神→十神、用神落点都在一起 | 只保留"用神五行（候选）"给 LLM，其余标注为 DEBUG |
| **六亲助力过于详细** | 每个大类都打印来源、强度话术，内容很长 | LLM 可能不需要这么详细的来源描述，可以简化或标注为 DEBUG |
| **原局/大运混杂** | 原局 section 里打印了大运转折点、用神互换、大运快照 | 移到大运 section，保持原局内容的纯粹性 |
| **天干五合、六合、半合** | 打印很详细，但 LLM 可能不需要 | 标注为 DEBUG，或者合并为"原局和谐信息"板块 |

### 8.2 建议改进方向

1. **立即改进**（标注）：
   - 使用**版本 1（括号标注法）**，给每个板块加标签
   - 明确哪些是 LLM 需要的，哪些是 DEBUG

2. **短期改进**（简化）：
   - 将"其他性格"折叠或标注为 DEBUG（因为 LLM 主要看主要性格和快速汇总）
   - 将"六亲助力"的来源描述简化（LLM 可能不需要这么详细）
   - 将"用神（五行→十神）"和"用神落点"标注为 DEBUG

3. **长期改进**（重构）：
   - 使用**版本 4（分层打印法）**，将原局内容分为 3 层
   - 将非原局内容（大运转折点、用神互换、大运快照）移到大运 section
   - 考虑后期做"免费版只显示 LLM 数据层，付费版显示全部"的裁剪
